## Post-processing of decoding pipeline for use in BWM paper

The files in this directory will allow you to convert the decoding pipeline output into a form that can be
plotted in the figures of the BWM paper.
The processing done in this step is primarily to read in the output summary tables of the decoding
pipeline and combine regions.  Specifically, the decoding pipeline outputs are sorted according to region
and session, but sessions of the same region need to be combined using fisher's method to plot per-region
results in the paper.

# set-up of directories
Copy the summary table files ("\*.csv" and "\*_xy.pkl" that result from the decoding pipeline) for use in 
this directory.  Paste the files into a sub-sub-directory called `decoding_results/summary/`.
Create a directory called `decoding_figures/`.

To process block summary tables, run the file
`
plot_decoding_block_panel.py
`
up to the end of the first block denoted by `# %%` (spyder notation).

# generate per-region (combined session) result files
Likewise, to process stimulus side, choice, feedback, wheel-speed, and wheel-velocity, run the first
block of the following files:
- `plot_decoding_stimside.py`
- `plot_decodoing_choice_panel.py`
- `plot_decoding_reward_panel.py`
- `plot_decoding_wheel-speed_panel.py`
- `plot_decoding_wheel-vel_panel.py`
.  

These files will each save per-region results that were used to generate figures in the BWM paper.
The swanson flatmats in the BWM pre-print were generated by 
Chris Langdon (figures 5, 6, 7, 8, 9, S8, S9, S13, S14, S17) using the resulting saved files.  

# plotting
If you desire to view the swanson flatmaps locally, you can generate them by running subsequent 
blocks of code in the above files.

The above files also save the weights that result from L1 regression for each variable.
The distribution of these weights are shown in figure S19 and can be generated by running
the script, `plot_hist_weights.py`.

You may also want to generate the decoding extra panels found in the main figures 5, 6, 7, 8, and 9.  There is one 
script which generates all of these extra panels for the figures, called `plot_extra_panels.py`.  It is not likely,
however, that you will need to do this because Michael Schartner has this code and was working on a script that 
plots all the analyses' extra panels at once.  In that case, the files used in this script should be sent to 
Michael. 

Next, you can generate the decoding bar plots found in figure S6 by running `plot_bwm_bars.py`.  
I previously shared a function with Yanliang Shi which creates these bar plots, `toyanliang_plotbars.py`
but this file is likely unnecessary.

Figure S15 has four panels which explain that wheel-velocity r2 scores for both the real and null targets are 
much lower than those of wheel-speed.  But because both are lower, the effect size and number of significant 
regions are comparable.  The panels that explain this can be generated by running `why_is_wheelvel_good.py`.
